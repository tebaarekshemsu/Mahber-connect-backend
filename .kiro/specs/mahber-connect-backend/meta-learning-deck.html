Motivation

What's wrong with
standard ML?

Standard Learning

One Task, Lots of Data

Train on thousands of examples of cats vs. dogs. Works beautifully — but start over for every new task.

Meta-Learning

Many Tasks, Few Examples

Given 5 examples of a never-seen class, classify it immediately. Not memorization — learning how to learn.

The goal: a model that, after seeing many tasks during training, can adapt to any new task from just a handful of examples.

Overview

Three Families of
Meta-Learning

Family I

Gradient-Based

Learn a good initialization. Adapt via gradient steps on the support set at test time.

MAML, Reptile, iMAML

Family II

Metric-Based

Learn an embedding space. Classify by distance to support set examples — no gradient steps.

ProtoNets, Matching Nets, Relation Nets

Family III

Model-Based

Learn a model with fast internal state. Absorb the task in one forward pass — no gradients needed.

All three ask: where do the task-specific "fast weights" live?

I
Family One

Gradient-Based
Meta-Learning

Learn an initialization so good that one gradient step adapts the model to any new task.

MAML — Core Idea

The Initialization Insight

Most initializations are equally distant from all task minima. MAML finds a special point — close to every task's minimum simultaneously.

Think geometrically: find the "centroid of task minima" in parameter space.

Not: find a model that's good.
But: find a model that's one step away from being good at anything.

Parameter space: θ*₁ (task 1 min) · θ_MAML · θ*₃ · θ*₂ (task 2 min) MAML θ is one gradient step from each θ*ᵢ.
  • Model-agnostic — works with any network trained by gradient descent
  • No architectural changes — pure training procedure
MAML — Algorithm

Two Nested Loops

① Inner Loop — Task Adaptation

For each task 𝒯ᵢ, take one gradient step on the support set from the shared θ:

θ'ᵢ = θ − α ∇_θ ℒᵢ(f_θ, 𝒟ᵗʳ)

α = inner learning rate. This adapts the model to task i.

② Outer Loop — Meta-Update

Evaluate adapted model on the query set. Update original θ:

ℒ_meta = Σᵢ ℒᵢ(f_{θ'ᵢ}, 𝒟ᵗᵉˢᵗ) θ ← θ − β ∇_θ ℒ_meta

β = meta learning rate. Gradient flows through θ'ᵢ back to θ.

for each meta-iteration: sample batch of tasks {𝒯ᵢ} for each 𝒯ᵢ: sample 𝒟ᵗʳ and 𝒟ᵗᵉˢᵗ # Inner loop θ'ᵢ = θ − α·∇_θ ℒ(𝒟ᵗʳ) # Outer loop (meta-update) θ ← θ − β·∇_θ Σᵢ ℒ(θ'ᵢ, 𝒟ᵗᵉˢᵗ) ↑ gradient through inner step!

The critical subtlety: computing ∂θ'ᵢ/∂θ requires a Hessian (2nd-order derivatives).

MAML Variants

FOMAML & Reptile

First-Order MAML

Drop the Hessian

Approximate the meta-gradient by ignoring second-order terms:

∇_θ ℒᵗᵉˢᵗ ≈ ∇_{θ'ᵢ} ℒᵗᵉˢᵗ

Valid when α is small (Hessian term is O(α)). Nearly as good empirically. Requires just one backward pass.

Reptile (Nichol 2018)

No Query Set Needed

Take k steps of SGD on task 𝒯ᵢ to get θ*ᵢ, then:

θ ← θ + ε(θ*ᵢ − θ)

Just move toward the adapted parameters. With k>1 steps, implicitly maximises within-task gradient coherence — second-order info sneaks back in!

Both FOMAML and Reptile optimise the same thing: maximise alignment between support-set and query-set gradients across tasks.

II
Family Two

Metric-Based
Meta-Learning

Learn an embedding space where similarity = proximity. Classification becomes nearest-neighbor search — no gradient steps at test time.

Metric-Based — Prototypical Networks

Classify by Prototype

Each class is represented by a single prototype — the mean of its support embeddings:

cₖ = (1/|Sₖ|) · Σ f_φ(xᵢ)

Classify queries by softmax over Euclidean distance to prototypes:

P(y=k|x̂) = softmax(−d(f_φ(x̂), cₖ))

Why Euclidean? Under Gaussian cluster assumptions, distance to the class mean is the Bayes-optimal classifier.

Embedding space ℝᵈ: 🟡 🟡 c₁ ● ← class 1 prototype 🟡 c₂ ● ← class 2 prototype 🔵 🔵 query x̂ → embed → nearest cₖ → class

1-shot case: prototype = the single example. ProtoNets ≡ Matching Nets in this regime.

Metric-Based — Other Methods

Matching & Relation Networks

Matching Networks (2016)

Soft Nearest Neighbor

Predict as a weighted sum over support labels using cosine attention:

P(y|x̂,S) = Σᵢ a(x̂,xᵢ)·yᵢ

Key innovation: context-aware embeddings — the embedding of each support example depends on the full support set (via biLSTM). The "right" features for a wolf depend on what else is in the episode.

Relation Networks (2018)

Learned Distance

Don't hand-choose Euclidean or cosine — learn the metric:

r = g_φ(concat(f(x̂), f(xₖ)))

A small network g_φ outputs a relation score in [0,1]. Can capture non-Euclidean, non-symmetric similarity structure.

The meta-learning in all three: episodic training forces embeddings to generalise to novel classes, not just memorise training ones.

III
Family Three

Model-Based
Meta-Learning

Build a model with enough internal capacity to absorb an entire task from a handful of examples — in a single forward pass.

Model-Based — MANN & SNAIL

Fast Weights in Memory

MANN — NTM (2016)

External Memory Matrix

An LSTM controller reads/writes to an external memory M ∈ ℝᴺˣᵈ via soft attention heads.

Labels arrive shifted by one step — forcing the model to bind inputs to labels in memory and retrieve them for future queries. Fast weights = memory slots.

SNAIL (2018)

TCN + Attention

The episode is a sequence: (x₁,y₁, x₂,y₂,…, x̂).

Temporal convolutions capture local dependencies. Attention allows global retrieval of any past example. Interleaving both handles both local dynamics and global memory.

Works for both classification and RL with the same architecture.

Core limit: the learning algorithm is bounded by the architecture's capacity. Complex tasks may need more adaptation depth than the network allows.

Unifying View

The Fast Weights Lens

All meta-learning families answer the same question differently: where does task-specific information live?

┌───────────────────────────────────────────────────────────────┐ │ META-LEARNING FAMILIES │ ├────────────────┬──────────────────┬────────────────────────────┤ │ GRADIENT-BASED │ METRIC-BASED │ MODEL-BASED │ ├────────────────┼──────────────────┼────────────────────────────┤ │ Fast weights = │ Fast weights = │ Fast weights = │ │ explicit θ'ᵢ │ prototypes cₖ │ memory M / LSTM state │ │ (gradient upd) │ (mean in ℝᵈ) │ (written per episode) │ │ │ │ │ │ Slow weights = │ Slow weights = │ Slow weights = │ │ init θ (MAML) │ encoder φ │ all model weights │ └────────────────┴──────────────────┴────────────────────────────┘

Hinton's "fast weights" intuition unifies them all: slow weights encode prior knowledge, fast weights encode the current task.

Modern Connection

Meta-Learning & Transformers

Large Transformers doing in-context learning are, at their core, model-based meta-learners:

  • Slow weights = transformer parameters (learned across vast task distribution during pretraining)
  • Fast weights = the key-value cache of the context (few-shot examples in the prompt)
  • Adaptation = attention over context — structurally identical to Matching Networks

GPT few-shot prompting = SNAIL at scale. The formalism makes this precise.

Prompt (support set): "wolf" → embedding "eagle" → embedding "fern" → embedding Query: "[new image]" → attend over context → predict class This IS metric-based meta-learning. The Transformer learned how to do it from billions of implicit tasks.

Gradient-Based

Learn where to start. MAML finds an init one gradient step from any task optimum.

Learn a geometry. ProtoNets collapses each class to a prototype; nearest prototype wins.

Learn an algorithm. MANN/SNAIL reads the task, writes to memory, predicts — no gradient needed.
